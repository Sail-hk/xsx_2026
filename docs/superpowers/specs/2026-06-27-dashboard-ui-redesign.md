# 习思想刷题程序 UI 重设计

日期：2026-06-27  
范围：`quiz_app` 首页与刷题主界面视觉和布局重设计  
目标：把当前页面从“松散的大卡片 + 留白过多”改成更像正式产品的双栏仪表盘，提升结构感、完成度和持续刷题时的操作稳定性。

## 1. 设计目标

当前页面的主要问题：

- 视觉重心不稳定，顶部、左侧、右侧统计区彼此割裂。
- 大块留白没有承担信息层级作用，看起来像“没做完”。
- 首页空状态过于空泛，缺少进入练习的明确引导。
- 模块入口虽然拆成了按题型和按章节，但仍然停留在“列表堆叠”层面，缺少产品化界面质感。

本次重设计的目标：

- 建立清晰的双栏结构：左侧导航，右侧工作台。
- 压缩无效留白，把空白变成有秩序的呼吸空间。
- 让“开始练习”“切模块”“看进度”“答题反馈”四类操作形成稳定视觉路径。
- 整体风格更像正式网页应用，而不是静态说明页。

## 2. 视觉方向

采用“现代仪表盘”方向，但保留适度的人文感，避免做成冷冰冰后台。

- 主视觉基调：深墨灰 + 暖铜橙 + 浅米白
- 左侧导航：深色面板，形成稳定锚点
- 右侧主区：浅色工作台，承载内容和题目
- 边界语言：明确卡片边框和层次，减少当前偏空、偏糊的玻璃感
- 字体气质：标题保留有一定辨识度，正文和功能信息强调清楚、耐读

不采用的方向：

- 不做大面积朦胧磨砂留白
- 不做纯展示型海报首页
- 不做过度彩色或学生作业感界面

## 3. 页面结构

### 3.1 总体布局

页面改为稳定双栏：

- 左栏：固定宽度导航区
- 右栏：弹性宽度主工作区

桌面端中，左栏始终可见；移动端改为纵向堆叠，导航区置顶。

### 3.2 左侧导航区

左侧导航区包含：

- 程序标题
- 简短副标题
- `按题型` 模块卡组
- `按章节` 模块卡组

设计要求：

- `按题型` 与 `按章节` 是两个独立模块块，不混在同一视觉容器里
- 章节区允许出现内部滚动，避免章节过多把页面整体拉长
- 当前激活模块需要有清晰高亮，且与 hover 状态区分明显

### 3.3 右侧工作区

右侧分为两种状态：

1. 默认欢迎态
2. 刷题进行态

#### 默认欢迎态

欢迎态不再是简单空白提示，而是一个有结构的工作台：

- 顶部总览条：展示题库量、单选、多选、判断数量
- 中部双入口卡：`按题型开始`、`按章节开始`
- 底部简要使用建议

目标是让用户一进入页面就有可点、可看的内容，避免“空白大盒子”。

#### 刷题进行态

刷题时右侧结构为：

- 顶部状态条：当前模块、题目进度、正确率、顺序/随机切换、重新开始
- 中部题目主卡：题型标签、来源、题干、选项、反馈
- 底部动作区：查看答案、提交、下一题

题目主卡必须成为右侧视觉中心。

## 4. 关键交互

### 4.1 模块切换

- 点击左侧任一模块后，右侧立即切换到该模块题目
- 当前激活项保持高亮
- 切换后工作区顶部状态条同步更新模块名

### 4.2 欢迎态入口

欢迎态中的快捷入口卡，本质上仍然触发左侧已有的两套模块逻辑，不新增数据逻辑，只新增更强的可视入口。

### 4.3 题目交互

- 单选、判断保持点击即判定
- 多选保持“选择后提交”
- 正确、错误、已选三种状态在视觉上明显区分
- 反馈区不单独悬浮，嵌入题目主卡内部

### 4.4 留白控制

新的留白规则不是“铺满”，而是：

- 导航区紧凑、稳定
- 工作区顶部信息集中
- 题目主卡上下有呼吸空间
- 默认态必须有内容填充，不允许出现当前那种大块无作用空白

## 5. 视觉细节要求

- 导航按钮更像仪表盘中的控制入口，而不是普通表单按钮
- 统计卡统一为同一尺寸语言，数字突出，标签克制
- 题干字号较大，但不再大到挤压整体版式
- 来源标签、题型标签弱化为辅助信息，不抢主标题
- 阴影更收敛，依赖层级和边框而不是大面积发光

## 6. 响应式规则

### 桌面端

- 双栏并排
- 左栏固定宽度
- 右栏自适应拉伸

### 平板端

- 左栏缩窄
- 统计区允许换行
- 顶部状态条允许分两行

### 手机端

- 改为单栏
- 导航区置顶
- 题目主卡保持最大宽度利用
- 动作按钮纵向堆叠，确保点击区域足够大

## 7. 实施边界

本次只做 UI 和布局重设计，不改下面这些逻辑：

- 题库数据结构
- 题型分类逻辑
- 章节分类逻辑
- 判题逻辑
- 随机/顺序模式逻辑

允许做的代码范围：

- `index.html` 结构调整
- `styles.css` 全量重构
- `app.js` 中仅限于支持新 DOM 结构、欢迎态内容和状态展示所需的小幅调整

## 8. 验收标准

- 页面首屏不再出现“空白大面板未被利用”的观感
- 左侧 `按题型` 与 `按章节` 明确分离
- 右侧存在明显的工作台结构，而不是几张散卡
- 刷题时题目区成为绝对视觉中心
- 桌面和手机端都能正常阅读和操作
- 整体完成度达到“正式网页应用”而不是“课程作业页面”
